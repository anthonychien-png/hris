## 1. Application Overview

A fully browser-based Enterprise CRM Suite built as a single React functional component. The application manages 11 entity types across Sales, Service, Marketing, and Analytics modules, with role-based access control, a profile switcher, and admin-only product management.

---

## 2. Modules & Features

### Sales CRM
- **Dashboard** — stat cards for pipeline value, open leads, active campaigns, and open service requests; high-priority service requests panel; recent leads table
- **Companies** — full CRUD; detail panel shows associated contacts, employees, and service requests
- **Contacts** — full CRUD; detail panel shows associated leads and service requests
- **Employees** — full CRUD; each employee must belong to one Company; detail panel shows leads owned, opportunities owned, and invoices owned
- **Leads** — full CRUD; each lead requires one assigned Employee (Owner); convert-to-opportunity action
- **Opportunities** — full CRUD; each opportunity requires one assigned Employee (Owner); multi-product association
- **Products** — view for all users; create, edit, inactivate/activate, and delete restricted to Admin User only; active/inactive status badge; inactive rows visually dimmed
- **Invoices** — full CRUD; each invoice requires one assigned Employee (Owner); dynamic line items linked to products; real-time total calculation

### Service
- **Service Requests** — ServiceNow/Siebel-style case management; auto-numbered SRQ-0001 format; one required Contact, optional Company, zero-to-many Products; priority colour-coded cards (Critical/High/Medium/Low); status workflow (Open → In Progress → Pending → Escalated → Resolved → Closed); filter by Status and Priority; 8 realistic ACME manufacturing seed records

### Marketing
- **Campaigns** — full CRUD; one-to-many Employee associations; hub view with drill-down to events and goals
- **Events** — full CRUD; one-to-many Employee associations; zero-to-many Lead associations; linked to parent Campaign
- **Goals** — full CRUD; linked to parent Campaign; progress bars with percentage completion

### Analytics — Reports
Seven bar chart reports, all computed from live data:
- Revenue by Month
- Revenue by Quarter
- Revenue by Year
- Service Requests by Company
- Service Requests by Contact
- Leads Generated by Campaign
- Leads Generated by Event

## 3. Access Control

### Admin User
- Full access to all modules
- Exclusive access to Product management (create, edit, inactivate, delete)
- Admin section visible in sidebar navigation with red shield icon
- Red "Admin Only" badge on Products page header

### Employee Profiles
- Read and write access to all modules except Product management
- Products page shows an orange lock banner explaining admin-only restriction
- No Edit, Delete, or Inactivate buttons visible on products
- Detail panel shows a lock message instead of action buttons

### Profile Switcher
- Clickable footer button in the sidebar
- Dropdown lists Admin User and all 6 employee profiles
- Each profile shows initials avatar, name, and title
- Active profile marked with a blue checkmark
- Switching profiles triggers a toast notification
- Clicking outside the dropdown closes it

## 6. Technical Architecture

- **Framework:** React 18 with hooks (`useState`, `useMemo`)
- **Styling:** Custom CSS injected via `<style>` tag; light mode palette
- **Fonts:** Plus Jakarta Sans (body), Lora (headings), JetBrains Mono (data values)
- **Charts:** Custom SVG bar charts — no external chart library
- **State:** In-memory client-side state; resets on page reload
- **Icons:** Inline SVG paths — no icon library dependency
- **Build Tool:** Create React App (`react-scripts 5.0.1`)
